# 🎧 DJ Track Downloader（クラウドWebアプリ）デプロイ手順

スマホから外出先でも使える、パスワード保護つきの Web アプリです。サーバーが裏で yt-dlp を動かし、完了した曲は **スマホの「ファイル」に保存** できます。

> このフォルダ（`scloud-web`）の中身一式をそのままデプロイします。
> 必要ファイル：`app.py` / `index.html` / `Dockerfile` / `requirements.txt` / `render.yaml`

---

## いちばん簡単な手順（Render・無料）

### 1. コードを GitHub に置く
1. GitHub で新しいリポジトリを作成（Private 推奨）。
2. このフォルダの中身を全部アップロード（ドラッグ&ドロップでOK）。

### 2. Render でデプロイ
1. https://render.com にサインアップ（GitHub 連携でOK）。
2. 「New +」→「Web Service」→ さっきのリポジトリを選択。
3. Runtime は **Docker** が自動検出されます。Plan は **Free**。
4. 「Environment」→「Add Environment Variable」で必ず設定：
   - **Key:** `APP_PASSWORD` / **Value:** あなたの好きなパスワード（自分専用の鍵）
5. 「Create Web Service」。数分でビルド完了。
6. 表示される URL（例 `https://dj-track-downloader.onrender.com`）をスマホで開く。

### 3. スマホで使う
1. URL を開く → パスワードを入力して「入る」。
2. URL か 曲名リストを貼って「ダウンロード開始」。
3. 完了した曲の **「保存」**（または「📦 全部まとめてZIP保存」）をタップ →
   iPhone なら共有メニュー →「"ファイル"に保存」。

> 💡 ホーム画面に追加：Safari で URL を開き「共有」→「ホーム画面に追加」。アプリのように使えます。

## 他のホスト

### Railway（https://railway.app）
1. New Project →「Deploy from GitHub repo」。
2. Dockerfile を自動検出。Variables に `APP_PASSWORD` を追加。
3. 生成された Public Domain をスマホで開く。

### Fly.io（CLI）
```bash
brew install flyctl
fly launch            # Dockerfile を検出。アプリ名を決める
fly secrets set APP_PASSWORD=あなたのパスワード
fly deploy
```

## 環境変数（任意で調整）
| 変数 | 既定 | 説明 |
|---|---|---|
| `APP_PASSWORD` | （必須）| ログインパスワード。未設定だとダウンロード不可（安全のため） |
| `MIN_DUR` / `MAX_DUR` | 90 / 900 | 曲名検索時に採用する長さ（秒） |
| `SEARCH_N` | 6 | 検索の上位何件まで試すか |
| `SLEEP_OK`/`SLEEP_MISS`/`COOLDOWN` | 8/4/45 | 待機時間（長いほどレート制限に強い） |

## 知っておくこと（正直な注意）
- **保存は一時的**：クラウドの無料プランはディスクが揮発性で、再デプロイ/スリープ復帰で `downloads` 内のファイルは消えます。**取得したらその場でスマホに保存**してください（曲はサーバーに溜めない設計）。
- **無料プランはスリープする**：Render Free は無アクセスで眠り、次回起動に数十秒かかります。
- **必ずパスワードを設定**：公開URLなので、未設定だと誰でも使えてしまいます。本アプリは未設定だとダウンロードをロックします。
- **規約・著作権**：公開サーバーの運用者（あなた）の責任になります。SoundCloud の利用規約・各楽曲の権利を尊重し、私的利用の範囲で。気に入った曲は購入を推奨します。
- ローカル確認：`APP_PASSWORD=test python3 app.py` → `http://localhost:8765`（要 yt-dlp / ffmpeg）。
